# Анализ устойчивости моделей компьютерного зрения к искажениям входных данных

Проект для исследования того, как искажения входных изображений (шум, размытие, сжатие и т.д.) влияют на предсказания моделей компьютерного зрения. MVP позволяет загрузить изображение, применить выбранное искажение и сравнить предсказания и метрики устойчивости через веб-интерфейс.

## Установка

pip install -r requirements.txt


Рекомендуется использовать Python 3.9–3.11 и виртуальное окружение.

## Запуск приложения

Из корня проекта:

python -m src.cv_robustness.app

Либо:

cd src/cv_robustness && python app.py


Откроется веб-интерфейс Gradio (по умолчанию http://127.0.0.1:7860). Загрузите изображение, выберите модель и тип искажения, задайте параметры (слайдеры 0–1) и нажмите «Анализировать».

## Структура проекта

- **`src/cv_robustness/`** — основной пакет:
  - **`models/`** — обёртки моделей (единый API `load()`, `predict()`):
    - `base.py` — абстрактный `BaseModelWrapper`;
    - `resnet_imagenet.py` — ResNet-50 на ImageNet (демо);
    - `torchvision_imagenet.py` — универсальная обёртка для моделей torchvision (ResNet-18, MobileNetV3, EfficientNet-B0 и др.).
  - **`distortions/`** — искажения изображений:
    - `base.py` — абстрактный `BaseDistortion` с `apply(image, **params)`;
    - `implementations.py` — гауссов шум, размытие по Гауссу, сжатие JPEG, яркость/контраст.
  - **`metrics.py`** — метрики устойчивости (совпадение класса, изменение уверенности, смена класса).
  - **`app.py`** — Gradio-интерфейс MVP.

## Как добавить свою модель

1. В `src/cv_robustness/models/` создайте класс, наследующий `BaseModelWrapper`.
2. Реализуйте `load()` (загрузка весов) и `predict(image: np.ndarray) -> dict` с ключами хотя бы `top_classes`, `top_labels`, `top_confidences`.
3. Зарегистрируйте модель в `app.py`: добавьте её в словарь `MODELS` (имя для UI → класс).

## Как добавить новое искажение

1. В `src/cv_robustness/distortions/` создайте класс, наследующий `BaseDistortion`.
2. Задайте `name`, `description` и `params_spec` (список параметров для UI).
3. Реализуйте `apply(self, image, **params) -> image`.
4. Импортируйте класс в `distortions/__init__.py` и добавьте экземпляр в словарь `DISTORTIONS` в `app.py`.

## Параметры искажений (слайдеры 0–1 в UI)

- **Гауссов шум**: Параметр 1 → sigma (сила шума).
- **Размытие по Гауссу**: Параметр 1 → размер ядра, Параметр 2 → sigma.
- **Сжатие JPEG**: Параметр 1 → качество (5–95).
- **Яркость и контраст**: Параметр 1 → яркость, Параметр 2 → контраст.

## Режим автоисследования и выгрузка в CSV

Во вкладке «Автоисследование» интерфейса Gradio можно:

- загрузить изображение;
- выбрать модель и тип искажения;
- задать **шаг по параметрам** (слайдер `Шаг по параметрам`).

Приложение перебирает **все комбинации** параметров выбранного искажения (полная решётка) с указанным шагом, для каждой комбинации:

- применяет искажение;
- считает предсказания модели до и после;
- вычисляет метрики устойчивости.

Результаты:

- сохраняются в CSV-файл в папке `results` в корне проекта (имя вида `auto_study_<искажение>_<timestamp>.csv`);
- первые строки показываются в таблице во вкладке «Автоисследование».

Откройте CSV-файл в Excel и постройте графики зависимости метрик (например, `confidence_delta`, `top1_match`) от параметров искажения.
